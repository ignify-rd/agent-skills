# Merge Postman Results into Test Case Excel

## What this skill does

Reads the auto-postman tool output (an `.xlsx` file + screenshots folder) and merges results
into the target test case Excel file. For each matched test case it:

1. Fills the **Actual Result** column: `Status: {code}\nResponse:\n{body}`
2. Evaluates **Pass / Fail / Blocked** using Claude AI (comparing expected vs actual)
3. Writes the verdict to the **Status** column
4. Replaces the **sample JSON** in the Expected Result with the real response JSON
   (the surrounding description text — "1. Check api trả về:", status line, etc. — stays untouched)
5. Creates/updates the **"Evidence"** sheet with embedded screenshot images

## Inputs the user must provide

| Input | Description |
|-------|-------------|
| **Source xlsx** | Auto-postman output, e.g. `api_results_20260415_091720.xlsx` |
| **Target xlsx** | Test case file to update |
| **structure.json** | Column mapping for the target file |

### structure.json format

This file is generated by the existing `extract_structure.py` tool in the card4-skills project.
The script uses only the fields it needs and ignores the rest (`templateFile`, `totalColumns`,
`columnWidths`, `sheetName`, etc.).

```json
{
  "templateFile": "template.xlsx",
  "sheetName": "...",
  "headerRow": 14,
  "dataStartRow": 15,
  "totalColumns": 21,
  "lastCol": "U",
  "columnMapping": {
    "externalId":      0,
    "testCaseName":    1,
    "preConditions":   2,
    "priority":        3,
    "steps":           4,
    "expectedResults": 6,
    "actualResult":    7,
    "bugId":           13,
    "status":          9
  },
  "columnWidths": [...]
}
```

**Fields used by this skill:**

| Field | Required | Purpose |
|-------|----------|---------|
| `testCaseName` | yes | **Match key** — matched against `API Name` from source |
| `expectedResults` | yes | AI reads for evaluation; JSON portion replaced with real response |
| `actualResult` | yes | Filled with `Status: {code}\nResponse:\n{body}` |
| `headerRow` | yes | Which Excel row contains column headers |
| `dataStartRow` | yes | First Excel row with test case data |
| `status` | **optional** | Filled with `PASS` / `FAIL` / `BLOCKED`. If missing, auto-detected by scanning the header row for "Status", "Trạng thái", "Result", "Pass/Fail". |

> The script always uses the **first sheet** of the target xlsx. `sheetName` in structure.json is ignored.

## How to run

```bash
python scripts/merge_postman_results.py \
  --source "D:\Code\auto-postman\results\api_results_20260415_091720.xlsx" \
  --target "path\to\test_cases.xlsx" \
  --structure "path\to\structure.json"
```

Optional flags:

| Flag | Effect |
|------|--------|
| `--output path.xlsx` | Save to new file instead of overwriting target |
| `--no-ai` | Use simple status-code comparison instead of Claude AI |
| `--api-key KEY` | Anthropic API key (or set `ANTHROPIC_API_KEY` env var) |

## Workflow when user asks to merge

1. **Confirm inputs**: ask for source xlsx, target xlsx, structure.json paths if not given
2. **Validate structure.json**: check that `testCaseName`, `expectedResults`, `actualResult`
   are present. If missing, scan the target header row and suggest values.
3. **Run the script**: use the command above
4. **Report results**: matched count, PASS/FAIL/BLOCKED breakdown, unmatched test cases
5. If many test cases are unmatched, compare API Name values between files to diagnose mismatches

## Evidence sheet

The script creates (or replaces) the "Evidence" sheet with:

- **Column A** — `ID`: test case name
- **Column B** — `Evidence`: screenshot image embedded directly in the cell

Screenshots are extracted directly from the embedded images in the source xlsx — no external
screenshot folder needed. Missing images are replaced with a `[No screenshot]` placeholder.

## Troubleshooting

**Test cases not matching**
The `API Name` in source must exactly match the `testCaseName` column value in the target,
including Vietnamese diacritics and special characters. Print both sets to compare:
```bash
python scripts/merge_postman_results.py --source ... --target ... --structure ... --dry-run
```

**AI evaluation not running**
Set env var `ANTHROPIC_API_KEY` or pass `--api-key`. Without a key the script falls back to
simple status-code comparison.

**openpyxl image error**
Install Pillow: `pip install pillow`
